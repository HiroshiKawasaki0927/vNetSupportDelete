# Chat Log: vNetSupportDelete リポジトリ作成と手順書整備

日付: 2026-04-02

## 概要

Power Platform 連携用仮想ネットワークサポートの削除手順をまとめたリポジトリ `vNetSupportDelete` を作成し、手順書を整備した。

## 実施内容

### 1. 環境構築

- GitHub CLI (`gh`) をインストール (`winget install GitHub.cli`)
- Git for Windows をインストール (`winget install Git.Git`)
- Personal Access Token を使用して GitHub に認証

### 2. リポジトリ作成

- `HiroshiKawasaki0927/vNetSupportDelete` リポジトリを作成（public）
- フォルダ構成を作成: AIDocs, assets, design, dev-envrionment, materials, scripts

### 3. README 作成・更新の経緯

1. 英語版 `README.md` を最初に作成
2. 日本語版 `README.jp.md` を作成
3. 画面操作ベースの手順に変更
4. 手順 1（サブネットインジェクション無効化）は **PowerShell でのみ実行可能** であることが判明
   - 公式ドキュメント: "You can remove an enterprise policy from an environment only through PowerShell"
   - `Disable-SubnetInjection` コマンドを使用
5. 手順 2（Enterprise Policy 削除）も Azure portal の検索で見つからないため、PowerShell + Resource Graph Explorer に変更
6. KQL クエリによる Enterprise Policy 確認手順を追加
7. 手順 2（確認）と手順 3（削除）を統合

### 4. トラブルシューティング

#### PowerShell 5.x での `-AsSecureString` エラー
- **原因**: モジュール v0.14.4 が PowerShell 5.x と非互換
- **対処**: PowerShell 7 をインストールして実行 (`winget install Microsoft.PowerShell` → `pwsh`)

#### `-PolicyArmId` パラメータが見つからないエラー
- **原因**: 正しいパラメータ名は `-PolicyResourceId`
- **確認方法**: `Get-Command Remove-SubnetInjectionEnterprisePolicy -Syntax`

#### サブネット委任解除時の `SubnetMissingRequiredDelegation` エラー
- **原因**: Enterprise Policy 削除後も `serviceAssociationLinks/PowerPlatformServiceLink` が残っている
- **対処**: 
  1. 数分～数十分待って再試行
  2. Enterprise Policy が本当に削除されたか Resource Graph で再確認
  3. `$subnet.ServiceAssociationLinks` で状態確認
  4. 解消しない場合は Microsoft サポートに連絡

### 5. 最終手順構成（5ステップ）

| 手順 | 内容 | 方法 |
|---|---|---|
| 手順 1 | サブネットインジェクション無効化 | PowerShell (`Disable-SubnetInjection`) |
| 手順 2 | Enterprise Policy 確認・削除 | Azure Resource Graph Explorer (KQL) + PowerShell (`Remove-SubnetInjectionEnterprisePolicy`) |
| 手順 3 | サブネット委任解除 | Azure ポータル |
| 手順 4 | サブネット削除 | Azure ポータル |
| 手順 5 | 仮想ネットワーク削除 | Azure ポータル |

## 使用したコマンド一覧

```powershell
# モジュールインストール
Install-Module -Name Microsoft.PowerPlatform.EnterprisePolicies
Import-Module Microsoft.PowerPlatform.EnterprisePolicies

# サブネットインジェクション無効化
Disable-SubnetInjection -EnvironmentId "<環境ID>"

# Enterprise Policy 確認
Get-SubnetInjectionEnterprisePolicy -SubscriptionId "<subscriptionId>" -ResourceGroupName "<resourceGroupName>"

# Enterprise Policy 削除
Remove-SubnetInjectionEnterprisePolicy -PolicyResourceId "<PolicyResourceId>"

# コマンド構文確認
Get-Command Remove-SubnetInjectionEnterprisePolicy -Syntax

# モジュールバージョン確認
Get-InstalledModule -Name Microsoft.PowerPlatform.EnterprisePolicies
Find-Module -Name Microsoft.PowerPlatform.EnterprisePolicies
```

```kql
// Enterprise Policy 一覧取得 (Resource Graph Explorer)
resources
| where type == "microsoft.powerplatform/enterprisepolicies"
| project name, kind, resourceGroup, location, subscriptionId, id, properties
```
